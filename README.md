## 06.2026

Структура Explore-With-Me разбита на микросервисную архитектуру.

# ExploreWithMe — Microservices Architecture

Система построена на Spring Boot и использует инфраструктурные компоненты:
- API Gateway
- Eureka Service Discovery
- Config Server
- OpenFeign для межсервисного взаимодействия
- Resilience4j для устойчивости к сбоям

## Архитектура

Система разделена на следующие микросервисы:

### 1. Event Service (События)
Отвечает за:
- создание и управление событиями
- публикацию и поиск событий
- фильтрацию и публичный API событий

### 2. Request Service (Заявки на участие)
Отвечает за:
- создание заявок на участие в событиях
- подтверждение и отклонение заявок
- подсчет количества подтвержденных участников

### 3. User Service (Пользователи)
Отвечает за:
- управление пользователями
- административные операции над аккаунтами

### 4. Comment Service (Комментарии)
Отвечает за:
- создание и получение комментариев
- модерирование комментариев

### 5. Gateway Service (API Gateway)
Отвечает за:
- единый вход в систему
- маршрутизацию запросов к микросервисам
- скрытие внутренней архитектуры системы от клиентов

### 6. Config Server
Отвечает за:
- централизованное хранение конфигураций
- выдачу настроек микросервисам

### 7. Discovery Server (Eureka)
Отвечает за:
- регистрацию сервисов
- динамическое обнаружение сервисов

## API Gateway маршруты

Все запросы клиентов проходят через Gateway (порт `8080`).

## Конфигурация

Конфигурации всех сервисов централизованы в Config Server.

## Устойчивость к сбоям

Для обработки отказов между сервисами используется:
- Resilience4j Circuit Breaker
- Retry механизмы
- fallback-методы в Feign-клиентах

## 01.2026

Реализована фича "Комментарии".

POST /users/{userId}/comments - Зарегистрированный пользователь может оставить комментарий к 
опубликованному событию.

PATCH /users/{userId}/comments/{commentId} - Пользователь может изменить свой комментарий, если он 
не опубликован. Если комментарий был отклонен администратором, он возвращается на рассмотрение.

DELETE /users/{userId}/comments/{commentId} - Пользователь может удалить свой комментарий в любом статусе.

GET /admin/comments - Администратор может сделать выборку комментариев с сортировкой по различным 
параметрам.

PATCH /admin/comments - Администратор может перевести комментарии в опубликованные или отклоненные 
по списку их id, если они находятся в статусе PENDING.

DELETE /admin/comments/{commentId} - Администратор может удалить любой комментарий.

GET /events/{id} - вместе с Событием возвращаются его опубликованные комментарии.

GET /users/{userId}/events/{eventId} - вместе с Событием возвращаются его опубликованные комментарии.

GET /admin/events - вместе с подборкой Событий возвращаются их опубликованные комментарии.
